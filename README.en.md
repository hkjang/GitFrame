## 🚀 What is GitFrame?

GitFrame is a local developer CLI that automatically runs your web project, validates its behavior using a browser scenario defined in YAML, and packages the entire run into **a demo video (MP4), subtitles (SRT), thumbnail, execution report, and Playwright Trace**.

### Key Features
- ✅ **Auto Project Detection** — Automatically classifies Node.js, Python, Go, Java, and Docker Compose projects and recommends build/run commands
- ✅ **Isolated Execution** — Builds and runs inside a temporary workspace, leaving your original project untouched
- ✅ **Playwright Automation** — Validates scenarios and records the screen simultaneously in a headless high-resolution browser
- ✅ **FFmpeg Video Rendering** — Merges intro card, browser recording, and outro card into a final MP4 automatically
- ✅ **Subtitle Support** — Each scenario step's `caption` field is automatically compiled into an SRT subtitle file
- ✅ **Diagnostic Artifact Packaging** — Generates HTML report, Playwright Trace, screenshots, and result JSON in one shot

## 🛠️ Installation

### Step 1: Install dependencies
```bash
npm install
```

### Step 2: Install Playwright browser binaries
```bash
npm run install-browsers
```

### Step 3: Install system libraries (Linux/WSL, first time only)
```bash
# Required system packages for Playwright
npx playwright install-deps

# Korean font to prevent subtitle rendering issues
sudo apt-get install -y fonts-nanum fonts-noto-color-emoji
```

### Step 4 (optional): Compile TypeScript
```bash
npm run build
```

## ⚡ Quick Start

### 1. Initialize GitFrame in your project
```bash
cd /path/to/my-web-project
npx gitframe init
```
This auto-generates `.gitframe/gitframe.yaml` and `.gitframe/demo.yaml` sample files.

### 2. Configure `gitframe.yaml`
```yaml
project:
  language: "go"                        # node | python | go | java | compose | static
  install: "go mod download"
  build: "go build -o server ./cmd/app"
  start: "./server"
  port: 8080
scenarioFile: ".gitframe/demo.yaml"
outputDir: "output"
video:
  width: 1440
  height: 900
  fps: 30
intro:
  title: "My Web App"
  subtitle: "GitFrame Demo"
  duration: 3
outro:
  title: "Thank You"
  subtitle: "Generated by GitFrame"
  duration: 2
captions: true
```

### 3. Define your browser scenario (`demo.yaml`)
```yaml
start_url: "http://localhost:8080"
steps:
  - id: step-1
    action: goto
    url: "/dashboard"
    caption: "Navigate to the main dashboard"

  - id: step-2
    action: pause
    milliseconds: 5000

  - id: step-3
    action: click
    selector:
      role: "button"
      name: "Load Data"
    caption: "Click the Load Data button"

  - id: final
    action: screenshot
    name: "finished"
    caption: "Scenario completed successfully"
```

### 4. Run the render pipeline
```bash
# Linux/WSL — specify shared library path
LD_LIBRARY_PATH=./lib/usr/lib/x86_64-linux-gnu node ./dist/cli/index.js render --output output

# macOS / standard environments
npx gitframe render --output output
```

### 5. Check results
```
output/
├── *.mp4            ← Final demo video
├── *-thumbnail.png  ← Video thumbnail
├── subtitles.srt    ← Subtitle file
├── report.html      ← HTML validation report
├── result.json      ← Result JSON
└── trace.zip        ← Playwright Trace
```

## 💻 CLI Commands

```text
gitframe
├── init      ← Initialize GitFrame config files in a project
├── inspect   ← Auto-detect project language/port and suggest commands
├── run       ← Start local web server in background + health check
├── record    ← Record manual browser actions and export as YAML scenario
├── test      ← Validate scenario in browser only (no video rendering)
├── render    ← Full pipeline: isolated build → playback recording → FFmpeg render
├── create    ← Clone a remote Git repo and run the full pipeline end-to-end
├── clean     ← Remove gitframe-* temp workspaces from /tmp
└── doctor    ← Diagnose local dependency status (Git/Node/Playwright/FFmpeg/Docker)
```

### `gitframe render` Internal Pipeline
```
[Copy Project] → [Install Deps] → [Build] → [Start Server] → [Health Check]
      ↓
[Playwright Scenario Playback] → [Screen Recording (WebM)] → [Save Playwright Trace]
      ↓
[FFmpeg: WebM → MP4 Transcode] → [Generate Intro/Outro] → [Compile Subtitles]
      ↓
[Merge Final MP4] → [Extract Thumbnail] → [Generate HTML Report] → [Cleanup]
```

### Supported Actions

| Action | Description |
| :--- | :--- |
| `goto` | Navigate to a URL |
| `click` | Click an element |
| `fill` | Type text into an input field |
| `pause` | Wait for specified milliseconds |
| `assertVisible` | Assert that an element is visible on screen |
| `assertText` | Assert that an element's text matches expected value |
| `waitFor` | Wait until an element appears |
| `screenshot` | Capture and save a screenshot |

### Locator Strategy (Priority Order)
> `testId` > `role` + `name` > `label` > `placeholder` > `text` > CSS

## ❓ Troubleshooting

| Symptom | Cause | Fix |
| :--- | :--- | :--- |
| Korean/CJK font garbled in video | Korean font not installed on Linux | `sudo apt-get install -y fonts-nanum` |
| Emoji appears as □ | Noto Color Emoji font not installed | `sudo apt-get install -y fonts-noto-color-emoji` |
| `libnspr4.so` error | Missing shared library in WSL environment | `LD_LIBRARY_PATH=./lib/usr/lib/x86_64-linux-gnu node ./dist/cli/index.js render` |
| `drawtext` FFmpeg error | FFmpeg built without font filter support | GitFrame automatically falls back to solid color card mode |
