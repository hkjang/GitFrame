# GitFrame × AI 에이전트 활용 가이드 🤖

> **이 문서는** GitFrame CLI와 AI 코딩 에이전트(Claude, Gemini, ChatGPT 등)를 함께 사용하여, 로컬 웹 프로젝트의 데모 영상을 **대화만으로** 완전 자동 생성하는 방법을 안내합니다.

---

## 🧭 핵심 개념: AI 에이전트가 GitFrame을 대신 실행한다

GitFrame CLI는 모든 단계가 명령어 기반으로 이루어지기 때문에, AI 에이전트(셸 실행 도구 + 파일 읽기/쓰기 도구를 갖춘)가 사람 대신 전 과정을 수행할 수 있습니다.

```
사용자 (자연어 요청)
    ↓
AI 에이전트 (프로젝트 분석 → gitframe.yaml 생성 → demo.yaml 생성 → render 실행)
    ↓
GitFrame (빌드 → 플레이백 → FFmpeg 렌더링)
    ↓
최종 결과물 (MP4 영상, 자막, 리포트)
```

**사용자가 해야 할 일은 딱 하나:**
> "이 프로젝트 데모 영상 만들어줘."

---

## 🔧 사전 준비 (최초 1회)

AI 에이전트에게 GitFrame을 처음 맡기기 전에, 로컬 환경에 아래가 설치되어 있어야 합니다.

```bash
# 1. GitFrame 저장소 클론
git clone https://github.com/hkjang/GitFrame.git /path/to/GitFrame

# 2. 의존성 설치
cd /path/to/GitFrame && npm install

# 3. Playwright 브라우저 설치
npm run install-browsers

# 4. 시스템 라이브러리 설치 (Linux/WSL)
npx playwright install-deps
sudo apt-get install -y fonts-nanum fonts-noto-color-emoji

# 5. TypeScript 컴파일
npm run build
```

## 💬 AI 에이전트에게 보내는 프롬프트 예시

### ✅ 기본 패턴: "이 프로젝트 영상 만들어줘"

가장 간단한 요청 방법입니다. 프로젝트 경로만 전달하면 AI 에이전트가 알아서 분석·설정·실행합니다.

```
/mnt/d/project/my-webapp 프로젝트로 데모 영상 만들어줘.
```

```
/home/user/projects/admin-dashboard 프로젝트 데모 영상 만들어서 제공해줘.
```

---

### 🎨 시나리오를 직접 지정하고 싶을 때

특정 화면이나 기능을 영상에 포함하거나 제외하고 싶을 때 사용합니다.

```
/mnt/d/project/my-webapp 프로젝트 데모 영상 만들어줘.
주요 화면은 대시보드, 사용자 관리, 설정 페이지 위주로 담아줘.
로그인 화면은 제외해도 돼.
```

```
/mnt/d/project/ecommerce 프로젝트로 영상 만들어줘.
각 화면마다 10초씩 머물고, 장바구니 추가 → 결제 흐름도 꼭 포함해줘.
```

인트로/아웃트로 타이틀이나 영상 길이를 제어할 수 있습니다.

```
/mnt/d/project/saas-platform 프로젝트 데모 영상 만들어줘.
인트로 타이틀은 "SaaS Admin Platform", 서브타이틀은 "Enterprise Dashboard Demo"로 해줘.
영상은 3분 이상 되게 만들어줘.
```

```
/mnt/d/project/api-gateway 프로젝트 영상 만들어줘.
각 페이지를 15초씩 보여주고, 한글 자막도 각 단계별로 설명 있게 넣어줘.
```

```
/mnt/d/project/project-a, /mnt/d/project/project-b, /mnt/d/project/project-c

```

### 🐛 오류가 생겼을 때 재시도 요청

AI 에이전트가 오류를 만났을 때 재시도하거나 원인을 분석하도록 요청할 수 있습니다.

```
이전에 만들다가 libnspr4.so 오류가 났어.
LD_LIBRARY_PATH 설정해서 다시 시도해줘.
```

```
영상에서 한글이 깨져 보여. 나눔 폰트 설치하고 다시 렌더링해줘.
```

```
이모지가 네모로 보여. 해결하고 재렌더링해줘.
```

## 🔄 AI 에이전트의 내부 작업 흐름

AI 에이전트는 다음 순서로 작업을 자동 처리합니다. 각 단계를 이해하면 요청을 더 정확하게 할 수 있습니다.

```
1. 프로젝트 구조 분석
   ├── 언어/프레임워크 감지 (Go, Node.js, Python 등)
   ├── 라우트/페이지 파악 (코드 분석 또는 README 참조)
   └── 포트 번호 확인

2. .gitframe/ 설정 파일 생성
   ├── gitframe.yaml 작성 (빌드/실행 명령, 해상도, 인트로/아웃트로)
   └── demo.yaml 작성 (화면별 goto + pause + caption 시나리오)

3. GitFrame render 파이프라인 실행
   ├── 격리된 임시 workspace에 프로젝트 복사
   ├── go mod download / npm install / pip install 등
   ├── 프로젝트 빌드 및 서버 구동
   ├── 헬스체크 (포트가 열릴 때까지 대기)
   ├── Playwright 브라우저 자동화 + 화면 녹화
   └── FFmpeg 비디오 렌더링 (인트로 + 본편 + 아웃트로)

4. 결과물 확인 및 링크 제공
   ├── output/demo.mp4 확인
   ├── 오류 발생 시 자동 진단 및 재시도
   └── 성공 시 파일 경로 및 요약 리포트 제공
```

## 📝 AI 에이전트가 생성하는 설정 파일 예시

### `gitframe.yaml` 예시 (Go 프로젝트)

```yaml
project:
  language: go
  install: "go mod download"
  build: "go build -o server ./cmd/app"
  start: "./server -transport http -addr 127.0.0.1:8080"
  port: 8080
  env: {}
scenarioFile: .gitframe/demo.yaml
outputDir: output
video:
  width: 1440
  height: 900
  fps: 30
intro:
  title: My App Platform
  subtitle: Go Web Server Demo
  duration: 3
outro:
  title: Thank You
  subtitle: Generated by GitFrame
  duration: 2
captions: true
```

### `demo.yaml` 예시 (3분 분량 18단계)

```yaml
start_url: "http://localhost:8080"
steps:
  - id: step-1-dashboard
    action: goto
    url: "/"
    caption: "메인 대시보드에서 전체 현황을 조회합니다"

  - id: wait-1
    action: pause
    milliseconds: 10000

  - id: step-2-users
    action: goto
    url: "/admin/users"
    caption: "사용자 관리 화면에서 계정 목록을 확인합니다"

  - id: wait-2
    action: pause
    milliseconds: 10000

  # ... (화면 수만큼 반복)

  - id: final-screenshot
    action: screenshot
    name: "finished-run"
    caption: "성공적으로 데모 시나리오를 완료하였습니다"
```

## ⚙️ 실행 명령어 참고 (AI 에이전트가 사용하는 명령)

AI 에이전트는 환경에 따라 아래 명령어를 자동 선택하여 실행합니다.

```bash
# Linux / WSL 환경 (공유 라이브러리 경로 지정 필요)
LD_LIBRARY_PATH=/path/to/GitFrame/lib/usr/lib/x86_64-linux-gnu \
  node /path/to/GitFrame/dist/cli/index.js render --output output

# macOS / 일반 환경
npx gitframe render --output output

# 빌드 전 TypeScript 컴파일
npm run build --prefix /path/to/GitFrame
```

## 💡 팁: 더 좋은 영상을 만드는 요령

### 1. 각 단계에 충분한 대기 시간 확보
SPA(React/Vue 등)는 화면 전환 후 데이터 로딩에 시간이 걸립니다. AI 에이전트에게 명시적으로 요청하세요.

```
각 페이지 전환 후 10~15초씩 머물러줘. 
데이터 로딩이 완료된 상태를 녹화해야 해.
```

### 2. 3분 이상 분량 확보
GitHub, YouTube 등에 올릴 데모 영상으로 적합한 최소 길이는 약 2~3분입니다.

```
영상 길이가 3분 이상 되도록 페이지마다 10초씩 머물고
화면이 충분히 많지 않으면 중요한 화면은 두 번 들러줘.
```

### 3. 자막(Caption) 활용
각 시나리오 단계의 `caption` 필드를 활용하면 영상에 한글 자막이 자동 삽입됩니다.

```
각 단계마다 해당 화면이 무엇을 하는 화면인지 
한국어로 설명하는 자막을 상세하게 써줘.
```

### 4. 라이브 데이터가 없어도 괜찮다
GitFrame은 실제 DB 연결 없이도 서버가 구동만 된다면 UI 화면을 녹화할 수 있습니다.

```
DB 연결 없이 서버를 띄울 수 있도록
환경변수를 적절히 설정해서 실행해줘.
```
