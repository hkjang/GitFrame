# GitFrame CLI 사용자 가이드 🎥

GitFrame은 웹 프로젝트를 자동으로 빌드 및 구동하고, 정의된 브라우저 시나리오(YAML)에 따라 동작을 테스트한 후, 그 전 과정을 고화질 **데모 영상(MP4), 자막(SRT), 썸네일, 실행 리포트, Playwright Trace** 등으로 패키징해 주는 개발자용 자동화 CLI 도구입니다.

---

## 📂 디렉터리 구조 (Directory Structure)

GitFrame을 초기화하면 프로젝트 루트에 `.gitframe` 폴더가 생성되며 다음과 같은 구조를 갖춥니다.

```text
현재-프로젝트-디렉터리/
├── .gitframe/
│   ├── gitframe.yaml      # GitFrame 글로벌 실행 및 영상 렌더링 설정 파일
│   ├── demo.yaml          # 플레이백으로 실행할 브라우저 자동화 시나리오 정의 파일
│   └── assets/            # 인트로/아웃트로 카드 배경 등 로컬 자산 보관용 폴더 (선택)
└── output/                # 렌더링 완료 후 결과물이 출력되는 디렉터리 (gitframe.yaml에서 변경 가능)
    ├── demo.mp4           # 최종 데모 영상 (인트로 + 브라우저 녹화 + 아웃트로 병합)
    ├── raw.webm           # 브라우저 원본 녹화 비디오 파일
    ├── thumbnail.png      # 비디오 대표 썸네일 이미지
    ├── result.json        # 실행 및 테스트 검증 상세 결과 리포트
    ├── scenario.json      # 실행에 활용된 YAML 시나리오의 JSON 변환 구조
    ├── report.html        # 시각화된 검증 결과 리포트 웹페이지
    ├── subtitles.srt      # 한글 자막 파일
    ├── logs/              # 서버 빌드, 구동, 렌더링 통합 진단 로그
    ├── screenshots/       # 시나리오 내 캡처(또는 실패 시점) 이미지 디렉터리
    └── trace.zip          # Playwright Trace Viewer 지원 추적 데이터 파일
```

## 🛠️ 설치 및 설정 방법 (Installation)

### 1. 의존성 패키지 설치
프로젝트 루트에서 아래 명령을 통해 의존성 라이브러리를 설치합니다.
```bash
npm install
```

### 2. 브라우저 바이너리 설치
Playwright가 구동하는 데 필요한 브라우저 엔진(Chromium 등)을 로컬에 다운로드합니다.
```bash
npm run install-browsers
```

### 3. 시스템 라이브러리 및 폰트 설치 (WSL / 리눅스 환경 필수)
리눅스 서버나 WSL 환경에서 브라우저가 실행(headless_shell)되려면 시스템 라이브러리와 한국어 폰트가 필요합니다.
```bash
# 필수 시스템 패키지 설치
npx playwright install-deps

# 한글 깨짐 방지를 위한 한국어 나눔 폰트 패키지 설치 (Ubuntu 기준)
sudo apt-get install -y fonts-nanum
```

#### 💡 WSL / Headless 환경 라이브러리 로드 트러블슈팅
시스템 권한 부족 등으로 브라우저가 실행 도중 `libnspr4.so` 등의 공유 라이브러리를 로드하지 못하고 종료될 때가 있습니다. GitFrame은 이를 해결하기 위해 자체 이식 가능한 `lib/` 경로를 내장하고 있으며, 실행 시 `LD_LIBRARY_PATH`를 지정해 줍니다.
```bash
LD_LIBRARY_PATH=./lib/usr/lib/x86_64-linux-gnu node ./dist/cli/index.js <command>
```

## ⚙️ 설정 파일 상세 가이드 (`gitframe.yaml`)

`.gitframe/gitframe.yaml`은 프로젝트 빌드/실행 정보와 최종 비디오 생성 옵션을 정의합니다.

```yaml
project:
  language: "go"               # 프로젝트 언어 유형 (node, python, go, java, compose, static 등)
  install: "go mod download"   # 프로젝트 의존성 설치 명령어
  build: "go build -o server ./cmd/app" # 프로젝트 빌드 명령어
  start: "./server"            # 프로젝트 서버 구동 명령어
  port: 8080                   # 모니터링할 웹 서버 포트 번호
  env:                         # 구동 시 주입할 추가 환경 변수
    DATABASE_URL: "sqlite://data.db"
    
scenarioFile: ".gitframe/demo.yaml" # 실행할 브라우저 자동화 시나리오 경로
outputDir: "output"                 # 결과 파일이 출력될 디렉터리 경로

video:
  width: 1440                  # 녹화 비디오 해상도 가로폭 (화질 개선을 위해 1440 권장)
  height: 900                  # 녹화 비디오 해상도 세로폭 (화질 개선을 위해 900 권장)
  fps: 30                      # 비디오 프레임 레이트

intro:
  title: "My Operations Platform"  # 인트로 카드 메인 타이틀
  subtitle: "System Demo Video"     # 인트로 카드 서브 타이틀
  duration: 3                      # 인트로 표시 시간(초)

outro:
  title: "Thank You"           # 아웃트로 카드 메인 타이틀
  subtitle: "Generated by GitFrame" # 아웃트로 카드 서브 타이틀
  duration: 2                      # 아웃트로 표시 시간(초)

captions: true                 # 자막(.srt) 파일을 비디오에 하드코딩(burn-in)할지 여부
```

## 📝 시나리오 파일 상세 가이드 (`demo.yaml`)

시나리오는 브라우저에서 차례대로 실행할 동작(Step)들을 정의합니다.

### 1. 기본 구조
```yaml
start_url: "http://localhost:8080" # 시작할 웹 애플리케이션 주소
steps:
  - id: step-1-login
    action: goto
    url: "/login"
    caption: "로그인 페이지로 이동합니다."
```

### 2. 지원하는 브라우저 동작 (Actions)

| 액션 이름 | 설명 | 필수 파라미터 | 예시 |
| :--- | :--- | :--- | :--- |
| **`goto`** | 특정 하위 경로(또는 절대 URL)로 이동 | `url` | `url: "/admin/dashboard"` |
| **`click`** | 지정한 요소를 마우스 클릭 | `selector` | `selector: "#submit-btn"` |
| **`fill`** | 텍스트 입력 필드에 값 기입 | `selector`, `value` | `value: "myPassword"` |
| **`pause`** | 지정한 시간(ms) 동안 대기 | `milliseconds` | `milliseconds: 5000` (5초) |
| **`assertVisible`**| 화면에 특정 요소가 활성화되었는지 검증 | `selector` | `selector: { role: "heading", name: "Welcome" }` |
| **`assertText`** | 특정 요소의 텍스트가 일치하는지 확인 | `selector`, `text` | `text: "Success!"` |
| **`waitFor`** | 특정 요소가 나타날 때까지 로딩 대기 | `selector` | `selector: ".loading-spinner"` |
| **`screenshot`** | 현재 화면 스크린샷 캡처 및 저장 | `name` | `name: "after-submit"` |

### 3. 요소를 찾는 방법 (Selector)
단순한 CSS Selector 외에도 접근성 기반의 지능형 선택자(Structured Selector)를 지원합니다.
GitFrame은 아래 우선순위대로 매칭되는 최적의 요소를 자동 탐색합니다.
> **우선순위**: Test ID (`testId`) > Role (`role`, `name`) > Label (`label`) > Placeholder (`placeholder`) > Text (`text`) > CSS

#### 예제:
* **CSS 방식**: `selector: "#username-input"`
* **Placeholder 방식**:
  ```yaml
  selector:
    placeholder: "아이디를 입력하세요"
  ```
* **접근성 Role 방식**:
  ```yaml
  selector:
    role: "button"
    name: "제출하기"
  ```
* **Test ID 방식 (HTML에 `data-testid`가 심어진 경우)**:
  ```yaml
  selector:
    testId: "dashboard-card-1"
  ```

## 💻 CLI 명령어 체계 (CLI Commands)

### 1. `gitframe init`
현재 작업 디렉터리에 샘플 설정 파일 및 디렉터리 구조를 생성합니다.
* **생성 파일**: `.gitframe/gitframe.yaml`, `.gitframe/demo.yaml`, `.gitframe/assets/`

### 2. `gitframe inspect`
프로젝트 구조를 정밀히 분석하여 어떤 언어(Node.js, Go, Python, Java, Docker Compose 등)로 짜였는지 유형을 감지하고, 추천되는 빌드/설치/구동 명령어 및 기본 포트를 제안합니다.

### 3. `gitframe run`
`gitframe.yaml` 설정을 기반으로 로컬 프로젝트 웹 서버를 백그라운드에 구동합니다. 포트가 열리고 정상 상태(Healthy)가 될 때까지 헬스체크를 수행하며, CLI 종료 시 포트 충돌 방지를 위해 백그라운드 자식 프로세스들을 일괄 안전하게 회수합니다.

### 4. `gitframe record`
헤디드 브라우저를 띄워 실시간으로 사용자의 마우스 클릭, 입력, 이동 등의 조작을 추적하여 자동으로 `.gitframe/demo.yaml` 테스트 시나리오 문법으로 기록 및 추출해 줍니다.
```bash
gitframe record --url http://localhost:8080 --output .gitframe/my-demo.yaml
```

### 5. `gitframe test`
작성한 시나리오가 웹에서 올바르게 실행되는지 렌더링 과정 없이 순수하게 브라우저만 띄워 고속 검증합니다. 시나리오 오동작 수정 시 유용합니다.
* `--headless` 옵션을 지정하지 않으면 실시간으로 브라우저가 조작되는 과정을 볼 수 있습니다.

### 6. `gitframe render`
가장 핵심적인 명령어로, 시나리오를 바탕으로 격리된 작업 환경에서 웹 애플리케이션을 구동해 화면을 레코딩하고, 자막 병합 및 인트로/아웃트로 카드를 결합하여 최종 `.mp4` 영상과 보고서를 패키징합니다.

### 7. `gitframe create <repoUrl>`
Shallow Clone 기법을 이용해 원격 Git 저장소를 복제한 뒤, 프로젝트 분석(`inspect`), 빌드 및 서버 실행(`run`), 검증(`test`), 비디오 최종 렌더링(`render`) 전 단계를 단 한 번의 명령어로 원스톱 자동화 처리합니다.
```bash
gitframe create https://github.com/example/my-project.git -o output-dir
```

### 8. `gitframe clean`
로컬 임시 폴더에 잔존하는 `gitframe-*` 격리 빌드 공간 및 Go 빌드 캐시를 말끔히 청소하여 디스크 용량을 확보합니다.

### 9. `gitframe doctor`
Git, Node.js, Playwright, FFmpeg, Docker 등 GitFrame 실행에 꼭 필요한 핵심 의존성 시스템 도구들이 설치되어 작동하는지 정합성을 자가 진단하여 결과 진단서를 출력합니다.

## ❓ 문제 해결 가이드 (Troubleshooting)

### Q1. 비디오 결과물에서 한글 폰트가 깨집니다.
* **원인**: 브라우저를 구동하는 리눅스 환경에 한글 폰트 패키지가 미설치된 상태입니다.
* **해결**: 아래의 폰트 설치 명령어를 수행한 후 다시 렌더링을 진행하세요.
  ```bash
  sudo apt-get update && sudo apt-get install -y fonts-nanum
  ```

### Q2. 웹페이지 내 이모지(Emoji)가 네모 상자(☐)로 보입니다.
* **원인**: 시스템 폰트에 이모지 폰트 라이브러리가 존재하지 않거나 브라우저 로드 시점에 인지하지 못했습니다.
* **해결**: Noto Color Emoji 등 폰트 의존성을 설치해 줍니다.
  ```bash
  sudo apt-get install -y fonts-noto-color-emoji
  ```

### Q3. FFmpeg 인트로/아웃트로 생성 시 `drawtext` 관련 오류가 나며 깨진 화질이 나옵니다.
* **원인**: 설치된 FFmpeg 바이너리가 `drawtext` 필터(폰트 엔진 연동)를 빌드 옵션에서 지원하지 않는 경우입니다.
* **해결**: GitFrame은 `drawtext` 필터 결함 시 자동으로 단색 카드(Solid Card) 모드로 폴백하여 오류 중단 없이 영상을 정상 결합하도록 예외 처리되어 있습니다. 최신 버전의 FFmpeg 패키지를 재설치하여 기능을 활성화할 수 있습니다.
