# GitFrame CLI — macOS Setup & Usage Guide 🍎

> **This guide is for macOS users only.** Unlike the Windows WSL/Ubuntu guide, you do **not** need to set `LD_LIBRARY_PATH` or use `apt-get`. Everything is handled through Homebrew and standard macOS tooling.

---

## 📋 macOS Requirements

| Item | Minimum |
| :--- | :--- |
| **macOS version** | Monterey (12.0) or later recommended |
| **Architecture** | Intel (x86_64) and Apple Silicon (M1/M2/M3) both supported |
| **Node.js** | v16.0.0 or later |
| **Homebrew** | Package manager (install if missing — see below) |
| **FFmpeg** | Install via Homebrew |
| **Git** | Xcode Command Line Tools or Homebrew |

---

## 🛠️ Step 1: Install Homebrew

Skip this step if Homebrew is already installed.

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

For Apple Silicon (M1/M2/M3), add Homebrew to your PATH after installation:
```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
```

---

## 🛠️ Step 2: Install Required Tools

### Node.js

**Option A: nvm (Recommended)**
```bash
# Install nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Restart your terminal, then install Node.js LTS
nvm install --lts
nvm use --lts

# Verify
node --version   # Should be v20.x or higher
npm --version
```

**Option B: Homebrew**
```bash
brew install node
node --version
```

### FFmpeg

On macOS, FFmpeg is installed via Homebrew — no custom builds needed. Homebrew's FFmpeg includes `libfreetype` (for the `drawtext` subtitle filter) by default.

```bash
brew install ffmpeg

# Verify drawtext support
ffmpeg -filters | grep drawtext
# Should output: "drawtext  Apply text drawing using the libfreetype library"
```

### Git (if not already installed)

```bash
# Option A: Xcode Command Line Tools (includes Git)
xcode-select --install

# Option B: Homebrew (latest version)
brew install git
```

---

## 🛠️ Step 3: Clone GitFrame and Set Up

```bash
# 1. Clone the repository
git clone https://github.com/hkjang/GitFrame.git ~/GitFrame
cd ~/GitFrame

# 2. Install Node.js dependencies
npm install

# 3. Compile TypeScript to JavaScript
npm run build

# 4. Install Playwright browser binaries
npm run install-browsers
```

> ✅ On macOS, you do **not** need to run `npx playwright install-deps` or set any `LD_LIBRARY_PATH`. Playwright detects macOS system libraries automatically.

---

## 🛠️ Step 4: Korean Font Check (for Subtitles)

macOS ships with Korean fonts (Apple Gothic, etc.) built in, so subtitle rendering typically works out of the box. If you see garbled characters, check and install as follows:

```bash
# List installed Korean fonts
fc-list :lang=ko

# If NanumGothic is missing, install it via Homebrew Cask
brew install --cask font-nanum-gothic

# Refresh font cache
fc-cache -fv
```

## ⚡ Quick Start

### 1. Initialize GitFrame in your project

```bash
cd ~/projects/my-web-project
node ~/GitFrame/dist/cli/index.js init

# Optional: link globally so you can use the "gitframe" command everywhere
cd ~/GitFrame && npm link
cd ~/projects/my-web-project && gitframe init
```

### 2. Configure `gitframe.yaml`

Edit `.gitframe/gitframe.yaml` to match your project:

```yaml
project:
  language: "go"                         # node | python | go | java | compose | static
  install: "go mod download"
  build: "go build -o server ./cmd/app"
  start: "./server"
  port: 8080
  env: {}
scenarioFile: ".gitframe/demo.yaml"
outputDir: "output"
video:
  width: 1440
  height: 900
  fps: 30
intro:
  title: "My Web App"
  subtitle: "GitFrame Demo"
  duration: 3
outro:
  title: "Thank You"
  subtitle: "Generated by GitFrame"
  duration: 2
captions: true
```

### 3. Write your scenario (`demo.yaml`)

```yaml
start_url: "http://localhost:8080"
steps:
  - id: step-1-home
    action: goto
    url: "/"
    caption: "Navigating to the home screen"

  - id: wait-1
    action: pause
    milliseconds: 10000

  - id: step-2-dashboard
    action: goto
    url: "/dashboard"
    caption: "Viewing the main dashboard"

  - id: wait-2
    action: pause
    milliseconds: 10000

  - id: final
    action: screenshot
    name: "complete"
    caption: "Scenario completed successfully"
```

### 4. Run the render pipeline

On macOS, run without any `LD_LIBRARY_PATH` prefix:

```bash
# Option A: Run directly with node
cd ~/projects/my-web-project
node ~/GitFrame/dist/cli/index.js render --output output

# Option B: After npm link — use global command
gitframe render --output output

# Option C: Using npx with GitFrame prefix
npx --prefix ~/GitFrame gitframe render --output output
```

### 5. Check your output

```
~/projects/my-web-project/output/
├── gitframe-xxxxxx.mp4   ← Final demo video
├── *-thumbnail.png       ← Video thumbnail
├── subtitles.srt         ← Subtitle file
├── report.html           ← HTML validation report
├── result.json           ← Result JSON
└── trace.zip             ← Playwright Trace
```

## 🔍 Auto-Detect Project Settings (`inspect`)

Let GitFrame analyze your project and recommend config values:

```bash
cd ~/projects/my-web-project
node ~/GitFrame/dist/cli/index.js inspect
```

Sample output:
```
✅ Detected language: Go
✅ Recommended install: go mod download
✅ Recommended build:   go build -o server ./cmd/app
✅ Recommended start:   ./server
✅ Detected port:       8080
```

## 🎙️ Record Browser Actions (`record`)

Open a headed browser, interact manually, and auto-generate a scenario file:

```bash
node ~/GitFrame/dist/cli/index.js record --url http://localhost:8080 --output .gitframe/demo.yaml
```

On macOS, the headed browser opens as a standard app window — making manual interaction straightforward.

## 🧪 Validate Scenario Without Rendering (`test`)

Quickly check your scenario runs correctly before committing to a full render:

```bash
node ~/GitFrame/dist/cli/index.js test
# or (after npm link)
gitframe test
```

## 🌐 One-Shot from Remote Repo (`create`)

Clone a GitHub repo and generate the demo video in one command:

```bash
node ~/GitFrame/dist/cli/index.js create https://github.com/example/my-project.git -o ./output
```

```bash
node ~/GitFrame/dist/cli/index.js clean
```

On macOS, temporary workspaces are created under `/tmp/gitframe-*`.

## ❓ macOS Troubleshooting

### Q1. `gitframe: command not found`

You haven't run `npm link` yet.

```bash
cd ~/GitFrame
npm link

# Or run via node directly
node ~/GitFrame/dist/cli/index.js <command>
```

### Q2. Gatekeeper security warning when Playwright launches Chromium

macOS Gatekeeper may block the Chromium binary downloaded by Playwright.

```bash
# Remove quarantine attribute from Playwright's cache directory
xattr -dr com.apple.quarantine ~/Library/Caches/ms-playwright/

# Or reinstall the browser
npx playwright install chromium
```

### Q3. Architecture mismatch on Apple Silicon (M1/M2/M3)

Ensure you're using an ARM-native Node.js build:

```bash
# Check your architecture
node -e "console.log(process.arch)"
# Should print: arm64

# If you see x64, reinstall Node.js using nvm on native ARM terminal
nvm install --lts
```

### Q4. FFmpeg reports missing `drawtext` filter

```bash
# Reinstall FFmpeg via Homebrew
brew reinstall ffmpeg

# Verify drawtext support
ffmpeg -filters 2>/dev/null | grep drawtext
# Expected: VS  drawtext  Apply text drawing using the libfreetype library
```

### Q5. Korean subtitle text is garbled

```bash
# Check available Korean fonts
fc-list :lang=ko | head -5

# Install NanumGothic if missing
brew install --cask font-nanum-gothic

# Refresh font cache
fc-cache -fv
```

### Q6. Port already in use error

```bash
# Find the process using the port
lsof -i :8080

# Kill the process by PID
kill -9 <PID>
```

When delegating GitFrame tasks to an AI agent on macOS, use macOS-style paths and omit WSL-specific instructions:

```
Generate a demo video for ~/projects/my-webapp.
GitFrame is at ~/GitFrame.
This is a macOS environment — no LD_LIBRARY_PATH setup needed.
```

## 📊 macOS vs WSL/Ubuntu: Key Differences

| Item | macOS | Windows WSL/Ubuntu |
| :--- | :--- | :--- |
| Package manager | Homebrew (`brew`) | apt-get |
| FFmpeg install | `brew install ffmpeg` | `sudo apt-get install ffmpeg` |
| `LD_LIBRARY_PATH` | ❌ Not needed | ✅ Required (`./lib/...`) |
| `playwright install-deps` | ❌ Not needed | ✅ Required |
| Korean font install | `brew install --cask font-nanum-gothic` | `sudo apt-get install fonts-nanum` |
| render command | `node ~/GitFrame/dist/cli/index.js render` | `LD_LIBRARY_PATH=... node ./dist/cli/index.js render` |
| Headed browser | Native app window | Requires X11 / WSLg |
| Temp dir | `/tmp/gitframe-*` | `/tmp/gitframe-*` |
