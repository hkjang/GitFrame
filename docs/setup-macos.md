# GitFrame CLI — macOS 설치 및 사용 가이드 🍎

> **이 문서는 macOS 사용자 전용 가이드입니다.** Windows WSL/Ubuntu 가이드와 다르게 `LD_LIBRARY_PATH` 설정이나 `apt-get` 패키지 설치가 필요 없으며, Homebrew를 통해 훨씬 간단하게 환경을 구성할 수 있습니다.

---

## 📋 macOS 환경 요건

| 항목 | 최소 요건 |
| :--- | :--- |
| **macOS 버전** | Monterey (12.0) 이상 권장 |
| **아키텍처** | Intel (x86_64) 또는 Apple Silicon (M1/M2/M3) 모두 지원 |
| **Node.js** | v16.0.0 이상 |
| **Homebrew** | 패키지 관리자 (미설치 시 아래 단계에서 설치) |
| **FFmpeg** | Homebrew로 설치 |
| **Git** | Xcode Command Line Tools 또는 Homebrew로 설치 |

---

## 🛠️ 1단계: Homebrew 설치

Homebrew가 없다면 먼저 설치합니다. 이미 설치된 경우 이 단계는 건너뜁니다.

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

Apple Silicon(M1/M2/M3) 사용자는 설치 후 아래 명령으로 PATH를 추가합니다:
```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
```

## 🛠️ 2단계: 필수 도구 설치

### Node.js 설치

**방법 A: nvm 사용 (권장)**
```bash
# nvm 설치
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 터미널 재시작 후 Node.js LTS 설치
nvm install --lts
nvm use --lts

# 설치 확인
node --version   # v20.x 이상이면 정상
npm --version
```

**방법 B: Homebrew 사용**
```bash
brew install node
node --version
```

### FFmpeg 설치

macOS에서 FFmpeg는 Homebrew로 간단히 설치합니다. WSL/Linux처럼 별도 빌드가 필요 없습니다.

```bash
brew install ffmpeg

# 설치 확인 (drawtext 필터 포함 여부 확인)
ffmpeg -filters | grep drawtext
```

> Homebrew의 FFmpeg는 기본적으로 `libfreetype`(drawtext 필터)이 포함되어 있어 자막 렌더링이 정상 작동합니다.

### Git 설치 (미설치 시)

```bash
# Xcode Command Line Tools 설치 (Git 포함)
xcode-select --install

# 또는 Homebrew로 최신 버전 설치
brew install git
```

## 🛠️ 3단계: GitFrame 클론 및 설정

```bash
# 1. 저장소 클론
git clone https://github.com/hkjang/GitFrame.git ~/GitFrame
cd ~/GitFrame

# 2. 의존성 설치
npm install

# 3. TypeScript 컴파일
npm run build

# 4. Playwright 브라우저 바이너리 설치
npm run install-browsers
```

> ✅ macOS에서는 `npx playwright install-deps`나 `LD_LIBRARY_PATH` 설정이 **필요하지 않습니다**. Playwright는 macOS에서 시스템 라이브러리를 자동으로 인식합니다.

---

## 🛠️ 4단계: 한글 폰트 확인 (자막 렌더링)

macOS는 기본적으로 한글 폰트(애플고딕, 나눔고딕 등)가 내장되어 있어 대부분의 경우 추가 설치가 필요 없습니다. 자막이 깨져 보일 경우 아래를 확인하세요.

```bash
# 시스템에 설치된 한글 폰트 목록 확인
fc-list :lang=ko

# 나눔 폰트가 없을 경우 Homebrew로 설치
brew install --cask font-nanum-gothic
```

## ⚡ 빠른 시작 (Quick Start)

### 1. 대상 프로젝트에 GitFrame 초기화

```bash
cd ~/projects/my-web-project
node ~/GitFrame/dist/cli/index.js init

# 또는 npm link 후 전역 명령어로 사용
cd ~/GitFrame && npm link
cd ~/projects/my-web-project && gitframe init
```

### 2. `gitframe.yaml` 설정

초기화 후 생성된 `.gitframe/gitframe.yaml`을 프로젝트에 맞게 수정합니다:

```yaml
project:
  language: "go"                         # node | python | go | java | compose | static
  install: "go mod download"
  build: "go build -o server ./cmd/app"
  start: "./server"
  port: 8080
  env: {}                                # 필요시 환경변수 추가
scenarioFile: ".gitframe/demo.yaml"
outputDir: "output"
video:
  width: 1440
  height: 900
  fps: 30
intro:
  title: "My Web App"
  subtitle: "GitFrame Demo"
  duration: 3
outro:
  title: "Thank You"
  subtitle: "Generated by GitFrame"
  duration: 2
captions: true
```

### 3. `demo.yaml` 시나리오 작성

```yaml
start_url: "http://localhost:8080"
steps:
  - id: step-1-home
    action: goto
    url: "/"
    caption: "메인 홈 화면으로 이동합니다"

  - id: wait-1
    action: pause
    milliseconds: 10000

  - id: step-2-dashboard
    action: goto
    url: "/dashboard"
    caption: "대시보드 화면을 확인합니다"

  - id: wait-2
    action: pause
    milliseconds: 10000

  - id: final
    action: screenshot
    name: "complete"
    caption: "시나리오가 완료되었습니다"
```

### 4. 데모 영상 렌더링

macOS에서는 `LD_LIBRARY_PATH` 설정 없이 바로 실행합니다.

```bash
# 방법 A: node 명령으로 직접 실행
cd ~/projects/my-web-project
node ~/GitFrame/dist/cli/index.js render --output output

# 방법 B: npm link 이후 전역 명령어로 실행
gitframe render --output output

# 방법 C: GitFrame 프로젝트 기준 npx 실행
npx --prefix ~/GitFrame gitframe render --output output
```

### 5. 결과 확인

```
~/projects/my-web-project/output/
├── gitframe-xxxxxx.mp4   ← 최종 데모 영상
├── *-thumbnail.png       ← 대표 썸네일
├── subtitles.srt         ← 자막 파일
├── report.html           ← HTML 검증 리포트
├── result.json           ← 결과 JSON
└── trace.zip             ← Playwright Trace
```

## 🔍 프로젝트 자동 감지 (`inspect`)

프로젝트 구조를 분석해서 언어, 포트, 실행 명령을 자동으로 추천합니다.

```bash
cd ~/projects/my-web-project
node ~/GitFrame/dist/cli/index.js inspect
```

출력 예시:
```
✅ 감지된 언어: Go
✅ 추천 install: go mod download
✅ 추천 build:   go build -o server ./cmd/app
✅ 추천 start:   ./server
✅ 감지된 포트:   8080
```

## 🎙️ 브라우저 동작 직접 녹화 (`record`)

수동으로 브라우저를 조작하면서 시나리오를 자동 생성할 수 있습니다.

```bash
node ~/GitFrame/dist/cli/index.js record --url http://localhost:8080 --output .gitframe/demo.yaml
```

macOS에서는 헤디드 브라우저가 일반 앱 창으로 열리므로 조작이 편리합니다.

## 🧪 시나리오 검증 (`test`)

렌더링 없이 시나리오만 빠르게 확인합니다.

```bash
node ~/GitFrame/dist/cli/index.js test
# 또는
gitframe test
```

## 🌐 원격 저장소 원스톱 처리 (`create`)

GitHub 등 원격 저장소를 Clone하여 영상까지 한 번에 생성합니다.

```bash
node ~/GitFrame/dist/cli/index.js create https://github.com/example/my-project.git -o ./output
```

## ❓ macOS 트러블슈팅

### Q1. `gitframe: command not found` 오류

`npm link`를 실행하지 않은 경우입니다.

```bash
cd ~/GitFrame
npm link

# 또는 node로 직접 실행
node ~/GitFrame/dist/cli/index.js <command>
```

### Q2. Playwright 브라우저 실행 시 보안 경고 (Gatekeeper)

macOS Gatekeeper가 Chromium 바이너리를 차단하는 경우입니다.

```bash
# Playwright 브라우저 경로에서 Gatekeeper 해제
xattr -dr com.apple.quarantine ~/Library/Caches/ms-playwright/

# 또는 브라우저를 재설치
npx playwright install chromium
```

### Q3. Apple Silicon(M1/M2/M3)에서 Rosetta 없이 실행 오류

ARM 네이티브 Node.js와 x86 바이너리 충돌 시:

```bash
# ARM 네이티브 Node.js 설치 확인
node -e "console.log(process.arch)"
# 출력이 'arm64'이면 정상

# Rosetta 필요 없이 ARM 네이티브로 Playwright 설치
npm run install-browsers
```

### Q4. FFmpeg `drawtext` 필터가 없다고 나올 때

```bash
# Homebrew FFmpeg를 최신으로 재설치
brew reinstall ffmpeg

# drawtext 지원 확인
ffmpeg -filters 2>/dev/null | grep drawtext
# "drawtext" 출력되면 정상
```

### Q5. 자막 한글이 깨질 때

```bash
# 시스템 한글 폰트 목록 확인
fc-list :lang=ko | head -5

# 폰트가 없으면 나눔고딕 설치
brew install --cask font-nanum-gothic

# 폰트 캐시 갱신
fc-cache -fv
```

### Q6. 포트가 이미 사용 중인 오류

```bash
# 사용 중인 프로세스 확인
lsof -i :8080

# 해당 PID 종료
kill -9 <PID>
```

## 🤖 AI 에이전트와 함께 사용하기 (macOS)

macOS에서 AI 에이전트에게 GitFrame 작업을 맡길 때는 아래와 같이 경로 형식을 macOS 스타일로 지정합니다.

```
# macOS 스타일 프롬프트 예시
~/projects/my-webapp 프로젝트로 데모 영상 만들어줘.
GitFrame은 ~/GitFrame에 있어.
macOS 환경이라 LD_LIBRARY_PATH 설정은 필요 없어.
```

```
# AI 에이전트용 실행 명령 (macOS 기준)
node ~/GitFrame/dist/cli/index.js render --output output
```

자세한 AI 에이전트 활용법은 [ai-agent-guide.md](ai-agent-guide.md)를 참고하세요.
